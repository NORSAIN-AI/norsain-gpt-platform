## Relevante faglige modeller og metoder

Dokumentet bygger på eller refererer til etablerte teknikker innen:
* **Chain-of-Thought reasoning** (Google Research, 2022)  
* **Self-Consistency** (Wang et al., 2022)  
* **Tree-of-Thoughts** (Yao et al., 2023)  
* **ReAct** (Reason+Act, 2022)  
* **Schema-guided prompting** (flere fagmiljøer, 2022–2024)  
* **Multimodal prompting** (Gemini, GPT-4V, Claude 3)  
* **Automatic Prompt Engineering (APE)** (evolusjonære tilnærminger innen LLM-optimalisering)

Teknikkene er bredt dokumentert i akademiske publikasjoner og industripraksis.

## Viktige faglige observasjoner gjennom dokumentet
* Prompt engineering er **ikke én teknikk**, men en samling strategier.  
* Det krever iterasjon, logging, versjonskontroll og testbarhet.  
* Mange teknikker komplementerer hverandre — lite fungerer isolert.  
* Strukturerte dataformater (JSON, schemas) er fundamentalt for systemintegrasjon.  
* LLM-er krever presisjon, tydelighet og klare regler for å gi pålitelige resultater.
